Intersection Types

  • combinations of two or more types, which are really useful for objects and parameters that must be implemented in more than one interface.

interface Animal { kind: string; } interface Person { firstName: string; lastName: string; age: number; } interface Employee { employeeCode: string; } let employee: Animal & Person & Employee = { kind: 'human', firstName: 'Jane', lastName: 'Smith', age: 20, employeeCode: '123' }

employee object has all the properties of Animal , Person , and Employee

Union Types

  • Create a new type that lets us create objects that have some/all of the properties of each type

interface Animal { kind: string; } interface Person { firstName: string; lastName: string; age: number; } interface Employee { employeeCode: string; } let employee: Animal | Person | Employee = { kind: 'human', firstName: 'Jane', lastName: 'Smith', age: 20 }

employee object of the Animal | Person | Employee type which means that it can have some of the properties of the Animal, Person, or Employee interfaces. Not all of them have to be included, but if they’re included, then the type has to match the ones in the interface
